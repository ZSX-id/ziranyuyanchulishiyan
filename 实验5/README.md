### 核心功能与操作说明
可以通过以下方式与助手交互：
**开始对话**：点击首页四张彩色卡片，或直接在底部输入框键入问题后按回车。
**图文提问**：点击输入框旁的图片按钮上传图片，可结合文字一起发送（模型为`qwen-vl-plus`）。
**查看回复**：AI的回答会以流式打字机效果呈现，并支持Markdown渲染（如表格、代码块）。代码块右上角有“复制”按钮。
**控制与切换**：
    生成过程中可点击**停止按钮**中断。
    点击**垃圾桶按钮**清除对话，返回首页。
    点击顶部**月亮/太阳图标**切换深色/浅色主题，主题状态会保存。

### 核心技术实现
1.这个项目采用前端分层架构，无后端依赖（直接对接阿里云百炼 API）
    UI 层：CSS 变量实现主题切换 + 响应式布局
    交互逻辑层：DOM 操作、事件监听、视图切换
    数据处理层：消息管理、格式转换、Markdown 渲染
    API 通信层：流式请求、AbortController 中断、SSE 解析
### 2.核心功能技术实现
1. 主题切换
   · 通过 CSS 变量定义深色/浅色两种配色方案。
   · 点击主题按钮时切换 body 的类名（dark-mode/light-mode），并将当前主题存入 localStorage，刷新页面后自动恢复。
2. 消息渲染与交互
   · 使用 messages 数组维护对话历史，每条消息包含角色（user/ai）和内容（纯文本或带图片的多模态数组）。
   · 动态创建 DOM：用户消息靠右，AI 消息靠左，并显示对应头像。
   · 对 AI 消息调用 marked.parse() 将 Markdown 转换为 HTML，再通过 highlight.js 对代码块进行语法高亮，并为每个代码块动态添加“复制”按钮（使用 navigator.clipboard）。
3. 流式响应（打字机效果）
   · 使用 fetch 发起 POST 请求，设置 stream: true。
   · 通过 response.body.getReader() 读取流式数据，按行解析 SSE 格式的 data: 块。
   · 每次接收到新的内容片段（delta.content）时，累加到当前 AI 消息的文本中，并重新渲染整个消息列表，从而实现逐字输出的视觉效果。
4. 图片上传与多模态消息
   · 点击图片按钮触发文件选择，使用 FileReader 读取图片为 Base64 字符串，预览并暂存。
   · 发送时将文本和图片组合成符合阿里云百炼要求的消息格式：[{ type: 'text', text }, { type: 'image_url', image_url: { url } }]。
   · 模型选用 qwen-vl-plus 以同时支持文本和图像理解。
5. 输入区控制
   · 监听输入框的 input 事件，当有内容（文字或图片）且未在加载时，显示发送按钮；否则隐藏。
   · 支持 Enter 发送，Shift+Enter 换行；点击停止按钮时通过 AbortController 中断请求。
6. 状态管理
   · 通过 isLoading 标志防止重复提交；currentAbortController 用于中断请求。
   · 清除对话时清空 messages 数组、隐藏消息区、重新显示首页欢迎卡片。
### 3.技术细节
1. 响应式布局
使用flex布局实现自适应（.app-container card-grid等）
max-width限制最大宽度，适配大屏；padding适配小屏
消息气泡max-width: 90%，避免超长文本溢出
2. 性能优化
流式渲染避免一次性渲染大量内容
scrollIntoView({ behavior: 'smooth' })实现平滑滚动
图片预览限制max-height/max-width，避免占用过多空间
3. 异常处理
API 请求失败时移除空白 AI 消息，避免界面异常
AbortError捕获用户主动中断，添加 “已中断” 提示
图片上传校验文件类型，非图片文件给出提示
